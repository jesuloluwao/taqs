## Codebase Patterns
- Convex indexes: use camelCase names (by_userId, by_entityId) — NOT snake_case (by_user_id was wrong on old schema)
- When updating Convex schema indexes, grep all .ts files for old index names (userMutations.ts had a stale reference)
- Stub out pages that call removed/changed mutations — avoids TS errors during schema overhaul iterations
- npm (not pnpm) is the package manager for this project: `npm run build`
- Build command: `npm run build` from project root (runs web build via workspace)

- Convex Actions: add `"use node";` at top of file to use Node.js APIs/npm packages (mutations cannot be in "use node" files — keep them separate)
- Cross-module internal calls from actions: use `(internal as any).moduleName.fnName` since _generated/api.d.ts isn't regenerated during dev
- PDF parsing in Convex: install `pdf-parse` in convex/package.json; use dynamic import inside action handler for CJS compatibility
- `makeFunctionReference` exists in convex/server but type assertions on `internal` are simpler for cross-module action→mutation calls
- Anthropic SDK in Convex: install in convex/package.json; use `await import('@anthropic-ai/sdk' as any)` + `mod.default ?? mod` pattern for compatibility
- `ctx.runAction` is available inside Convex actions to call other actions (useful for chaining async pipelines)
- AI failure in import pipeline: wrap `ctx.runAction(aiCategorise...)` in try/catch so import job completes even if AI is unavailable

  - Bank detection from PDF text content (GTBank, Zenith, Access, generic fallback)

  - Generic fallback: dateRe + amountRe pattern matching with CR/DR direction inference

- externalRef: SHA-256 hash of date|description|amount|direction (truncated 32 chars)
- Error handling: parse errors → setJobFailed; insert errors → setJobFailed with partial stats
- Partial parse resilience: try/catch per row — successful rows imported even if others fail
- `npm run build` passes (tsc + vite)
- **Learnings for future iterations:**
  - Convex `"use node"` files: ONLY export `action` functions — mutations/queries can't be in the same file
  - Internal function cross-file calls from actions: `(internal as any).module.fn` bypasses stale generated types

  - convex/package.json deps need `npm install` run in `/convex` directory separately
  - Amount storage convention: ALL amounts stored in smallest unit (kobo = Naira × 100) for integer math
